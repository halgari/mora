# Mora Integration Test Harness

## Goal

Verify that Mora produces equivalent runtime patches to SkyPatcher by comparing in-memory form state dumps from Skyrim. A standalone SKSE plugin (MoraTestHarness.dll) exposes a TCP command interface to dump form data on demand. A Python script drives the workflow: connect, dump, diff.

## Architecture

### mora_rt.lib (refactored from mora_rt.bc)

The existing `mora_rt` bitcode is refactored into a Windows static library compiled with `-flto` for LTO compatibility. This library is the shared foundation for both the patcher DLL and the test harness DLL.

**Existing code (moves into .lib):**
- CRC32 hashing (`src/rt/crc32.cpp`)
- BSTHashMap walker (`src/rt/hashmap_lookup.cpp`)
- Form field writers (`src/rt/form_ops.cpp`)
- Keyword mutation (`mora_rt_add_keyword`)

**New code (added to .lib):**
- Form dumper: reads weapon form fields (damage, value, weight, name, keywords) via the existing ABI shim, serializes to JSONL
- TCP command listener: Winsock2-based, single-threaded accept loop on a background thread, newline-delimited text protocol
- INI config reader: parses port and dump path from `MoraTestHarness.ini`

### MoraRuntime.dll (patcher â€” generated by `mora compile`)

No change to what it does: applies patches on DataLoaded. Build pipeline change:
- `DLLBuilder` drops the explicit `lto_merge()` step
- lld-link receives both the generated `.obj` (LTO bitcode) and `mora_rt.lib` (LTO bitcode)
- LTO happens at link time â€” same constant-folding wins, cleaner pipeline

### MoraTestHarness.dll (standalone SKSE plugin)

A small SKSE plugin with no patch logic. On DataLoaded:
1. Reads `MoraTestHarness.ini` for port and dump path
2. Starts TCP listener thread
3. Waits for commands

Links against `mora_rt.lib` for form reading, dumping, and TCP code. Cross-compiled on Linux targeting x86_64-pc-windows-msvc, same as all other Windows targets.

Source: `src/harness/harness_plugin.cpp` â€” SKSE entry points + DataLoaded handler.

The harness resolves Skyrim's global `BSTHashMap<FormID, TESForm*>` via the Address Library (same ID 514351 used by the patcher). The Address Library `.bin` file must be present in the Skyrim data directory (standard for any SKSE setup).

## TCP Protocol

Single TCP connection at a time. Newline-delimited text commands, JSON responses.

### Configuration

`MoraTestHarness.ini`, read from the same directory as the DLL (discovered via `GetModuleFileName`):
```ini
[General]
port=9742
dump_path=Data/MoraCache/dumps
```

### Commands (v1)

| Command | Response | Effect |
|---------|----------|--------|
| `status` | `{"ok":true,"forms_loaded":true}` | Confirms harness is ready |
| `dump weapons` | `{"ok":true,"file":"<path>","count":1234}` | Dumps all weapon forms to JSONL |
| `quit` | `{"ok":true}` | Closes listener |

### Dump Format

One JSON object per form, newline-delimited (JSONL). Sorted by FormID for deterministic diffs.

```json
{"formid":"0x00012EB7","name":"Iron Sword","damage":7,"value":25,"weight":9.0,"keywords":["0x0001E711","0x0006BBE3"]}
```

- FormIDs and keyword IDs as hex strings for readability
- Keywords sorted by FormID within each form
- Floating point values preserved at full precision

## Build System (xmake)

### New/changed targets

- `mora_rt` â€” changes from bitcode compilation to static library (`static`), compiled with `-flto`, target `x86_64-pc-windows-msvc`
- `mora_test_harness` â€” new target, shared library (DLL), links `mora_rt.lib`, cross-compiled
- `mora` CLI â€” `DLLBuilder` link step updated to link against `mora_rt.lib` instead of LTO-merging bitcode

### LTO flow

Both `.obj` (from LLVM IR codegen) and `mora_rt.lib` contain LTO bitcode. lld-link performs LTO at link time. The explicit `DLLBuilder::lto_merge()` method is removed.

## Test Workflow

### Python test driver (`tools/test_harness.py`)

The script handles TCP connection, command sending, file collection, and diffing. It does not launch Skyrim â€” the game is started manually or via mod manager.

**Capture:**
```bash
python tools/test_harness.py capture --tag skypatcher --commands "dump weapons"
# Connects to localhost:9742
# Sends "dump weapons", waits for response
# Copies dump file to test_data/snapshots/weapons_skypatcher.jsonl
```

**Compare:**
```bash
python tools/test_harness.py compare --expected skypatcher --actual mora
# Diffs the two JSONL files
# Exit 0 if identical, exit 1 with diff output
```

### End-to-end test sequence

1. **One-time:** Deploy `MoraTestHarness.dll` + `.ini` to Skyrim SKSE plugins
2. **Baseline capture:** Launch Skyrim with SkyPatcher configs, run `capture --tag skypatcher`
3. **Commit snapshot:** `test_data/snapshots/weapons_skypatcher.jsonl` becomes the expected baseline
4. **Mora capture:** Translate SkyPatcher configs to Mora, compile with `mora compile`, launch Skyrim with Mora DLL + harness, run `capture --tag mora`
5. **Compare:** `compare --expected skypatcher --actual mora` â€” diffs should be empty

## Scope (v1)

- Weapon forms only: formid, name, damage, value, weight, keywords
- Single `dump weapons` command
- JSONL file output with plain diff comparison
- No interactive querying, no filtered dumps, no other form types

## Future extensions

- `dump armor`, `dump npcs`, etc. as form types are added
- `dump <formid>` for single-form inspection
- Filtered dumps (`dump weapons --with-keyword 0x0001E711`)
- Semantic diff in Python (floating point tolerance, keyword set comparison)
- Automated game launch via Steam/MO2 CLI
